# self-improve

セッションでの訂正・手戻り・暗黙の期待を、再発防止の形で永続化するスキル。

## モード判定

- 引数なし or "review": **inboxモード** — 提案ボックスをレビュー
- "live" or 会話中の特定の学びを指されている: **liveモード** — 現在の会話を直接分析

## inboxモード手順

1. `~/.claude/self-improve/proposals/*.md` を全て読む。0件なら「提案はありません」と報告して終了
2. 提案を反映先ごとにグルーピングして要約をユーザーに提示する。重複・矛盾する提案は統合する
3. 各提案について AskUserQuestion で「反映 / スキップ / 修正して反映」を確認する（multiSelectで一括確認してよい）
4. 承認されたものを反映する（下記「反映ルール」参照）
5. 処理済み提案は `~/.claude/self-improve/archive/` に移動する（スキップ含む。frontmatterの `status:` を applied / skipped に更新）

## liveモード手順

1. 現在の会話を振り返り、以下を抽出する:
   - ユーザーからの訂正・指摘（Claudeのやり方をユーザーが直した箇所）
   - 手戻り・探索の空振り・誤った前提
   - ユーザーの好み・暗黙のルール
2. それぞれ反映先を分類して提案し、ユーザー確認のうえ反映する

## 反映ルール（重要）

| 分類 | 反映先 | 基準 |
|------|--------|------|
| 全プロジェクト共通のプロセス・好み | `~/.claude/CLAUDE.md` | プロジェクト名や特定コードに依存しない |
| プロジェクト固有の知識・ルール | 対象リポジトリの `CLAUDE.md` | リポジトリ内で完結する話 |
| スキル実行中の失敗・非効率 | 対象スキルの `SKILL.md` | 可能なら `skill-evaluator` スキルの手法に従う |
| 事実（ユーザー情報・進行中の作業） | memory ディレクトリ | ルールではなく事実 |

### ガードレール

- **CLAUDE.md は簡潔に保つ**。追記の前に既存項目と重複していないか確認し、重複なら既存項目を具体化する形で統合する。1つの学びは原則1〜2行
- 全体が長くなりすぎたら（目安: グローバルCLAUDE.mdで80行超）、追記ではなく既存内容の整理・統合を提案する
- セッション限りの話・一時的な事情は反映しない
- **既存の指示を削除・弱体化する変更は必ず理由とdiffを見せて個別確認する**
- git管理下のプロジェクトCLAUDE.mdを変更した場合は、変更した旨と diff を報告する（コミットはユーザー指示がある場合のみ）

## 完了報告

- 反映した項目 / スキップした項目を一覧で報告する
- 反映先ファイルは `path:line` で参照する
