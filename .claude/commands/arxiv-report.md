## このスキルについて

arxiv論文のURLを入力すると、以下を含む詳細な日本語レポートをMarkdown形式で生成します：

1. **詳細な論文解説**（関連研究のarxiv URL付き、論文中の図表をすべて含む）
2. **落合フォーマット**でのまとめ
3. **GitHub実装の解析**（コード抜粋付き）

処理には約15分程度かかるため、バックグラウンドで実行されます。

## タスク

### 手順1: 初期確認

1. ユーザーから提供されたarxiv URLを確認
2. URLからarxiv IDを抽出（例: `https://arxiv.org/abs/2501.12345` → `2501.12345`）
3. ユーザーに開始を通知

### 手順2: バックグラウンドエージェントの起動

**すぐに**Taskツールを使ってバックグラウンドで処理を開始してください：

```
Taskツールのパラメータ:
- subagent_type: "general-purpose"
- run_in_background: true
- description: "arxiv論文詳細レポート生成"
- prompt: 下記の詳細プロンプトを使用
```

#### バックグラウンドエージェント用のプロンプト:

```
arxiv論文 [arxiv_id] の詳細な日本語レポートを生成してください。

**重要**: 15分以内に完了するため、効率を重視してください：
- 複数のWebFetch/WebSearchは並列実行
- 関連研究は5本程度に絞る
- GitHub調査は見つからなければスキップ
- 図表は主要なもの5-7個程度

## 実行内容

### フェーズ1: 論文情報の取得（5分）

**効率的に並列実行してください**

1. WebFetchで `https://arxiv.org/abs/[arxiv_id]` から以下を抽出：
   - タイトル（原題と日本語訳）
   - 著者と所属
   - 発表日、カテゴリ
   - アブストラクト
   - GitHub URLやプロジェクトページ

2. WebFetchで `https://arxiv.org/pdf/[arxiv_id].pdf` から詳細分析：
   - 全セクションの内容
   - 提案手法の詳細
   - 実験設定と結果
   - **主要な図表を特定し、ページ番号を記録**（全部でなく重要なもの5-7個程度）
   - 引用文献リスト

### フェーズ2: 関連研究の調査（5分）

**効率重視：5本程度に絞る**

WebSearchで関連arxiv論文を調査：

1. `"[キーワード] arxiv [年]"` で2-3本
2. `"[第一著者名] arxiv"` で1-2本
3. 論文中の重要引用から1-2本

各論文：タイトル、URL、簡潔な関係性のみ記録

### フェーズ3: GitHub実装の調査（3分）

**見つからない場合はスキップ**

1. フェーズ1で見つかったGitHub URLにアクセス
2. なければ簡単に検索（1回のみ）
3. 見つかった場合のみ：
   - README確認
   - 主要コード1-2ファイルを特定
   - 重要部分50-100行程度を抽出

### フェーズ4: レポート生成（2分）
```
